## What This Template Does {#what}

The Engagement Strategy Map plans how to move users from Casual → Core → Power engagement states by:
1. Defining the **mindsets** that explain *why* users are in each state
2. Identifying the **signals** that surface when users are ready to move
3. Designing the **interventions** (real estate, message, moments) that drive the transition

It covers four levers for increasing engagement: **Adding Use Cases**, **Increasing Feature Usage**, **Increasing Frequency**, and **Increasing Intensity**. You don't need all four — choose the one most relevant to your current engagement gap.

---

## The Three Engagement States {#states}

Before building the map, define what each state means for your product:

| State | Definition | Retention Metric |
|-------|-----------|-----------------|
| **Casual** | Uses product for one limited use case or infrequently | [DAU/WAU/MAU — match natural frequency] |
| **Core** | Regular user, primary use case well established | Same metric, higher frequency |
| **Power** | Multi-use-case, high-frequency, deeply integrated into workflow | Same metric, highest frequency + breadth |

*Example (GrubHub): Casual = 1 order/week ("Friday tired" users). Core = 2–3 orders/week ("alternative to cooking"). Power = daily user across multiple meal types.*

### Part 4: Increasing Frequency

Use this section when users have the right use case and features but don't return often enough. Design habits loops that trigger return visits.

A habit loop has three parts: **Trigger → Action → Reward**. Design one loop per natural trigger.

| Field | Loop 1 | Loop 2 |
|-------|--------|--------|
| **Loop Name** | [Name] | [Name] |
| **Trigger** | What external or internal signal prompts the return visit? (e.g., "New message received", "Weekly report ready", "Teammate requested review") | |
| **Channel** | How does the trigger reach the user? (push, email, in-app, SMS) | |
| **Action** | What does the user do when they respond to the trigger? | |
| **Reward** | What value does the user receive that makes the loop worth repeating? | |
| **Loop Frequency** | How often does this loop fire? (daily, weekly, event-driven) | |

**PNIP Check** — Before adding notification/incentive/people triggers, ask:
- Can the **Product** itself create the trigger without intervention? (best)
- If not, does a **Notification** (push/email) make sense?
- If not, does an **Incentive** (badge, streak, reward) make sense?
- If not, can a **Person** (teammate, customer success) trigger the return?

## Example: GrubHub (Illustrative) {#grubhub}

*Illustrative example for educational purposes. Does not reflect actual GrubHub data.*

### Mindset Comparison Matrix

| State | Mindset | Description | Transition Success | Transition Failure |
|-------|---------|-------------|-------------------|--------------------|
| **Casual** | Friday Tired | "GrubHub is for when I'm exhausted and don't want to cook" | Discovered GrubHub for a non-Friday occasion (lunch at work); experienced same convenience | Tried it during a non-tired moment, compared price to cooking, decided not worth it |
| **Casual** | Pizza Night | "GrubHub is only reliable for pizza — I use Seamless for everything else" | Found a great non-pizza restaurant through browse; had a good first experience | Ordered a non-pizza item, it was late or wrong, reinforced the belief |
| **Core** | Weeknight Alternative | "GrubHub is what I use when I don't have time to cook on weeknights" | Started using for lunch too when work got busy | Seamless promo pulled them away mid-week |
| **Core** | Convenience Hub | "GrubHub saves me time because it has my preferences and payment saved in one place" | Invited a partner/roommate; shared orders increased utility | Realized DoorDash had more restaurants in their neighborhood |
| **Power** | Daily Optimizer | "GrubHub is the easiest way to handle every meal where I don't cook" | Discovered group ordering; now orders for team at work | |

### Adding Use Cases — Lunch at Work

| Field | Value |
|-------|-------|
| **Use Case** | Ordering lunch at work (Casual users currently only order Friday dinner) |
| **Signal** | User hasn't placed an order between Monday–Thursday AND has placed ≥3 Friday orders in past month |
| **# with Signal** | 120,000 users |
| **Real Estate** | In-app banner on Thursday evening (contextual timing) |
| **Message** | "Your Friday GrubHub is ready — but did you know 2M people order lunch at work through us? Save time tomorrow." |
| **Setup Moment** | User sets a "work address" in their account |
| **Aha Moment** | First weekday lunch order delivered on time to work |
| **Habit Moment** | 3 weekday lunch orders within 14 days (3a14) |

## How to Use This Template {#how-to-use}

### For Reviews (growth-review)
1. Start with the Mindset Comparison Matrix — ask the user about qualitative research or infer from engagement cohort data
2. Identify which of the four levers is the primary engagement gap (use case? feature? frequency? intensity?)
3. Fill in the relevant section — the funnel from Signal to Habit shows exactly where users drop off
4. Map drop-offs to specific product/notification/incentive interventions

### For Execution (growth-execution)
1. Choose one lever and one specific intervention to build (don't build all four at once)
2. The Signal → Real Estate → Message spec is your feature spec for the intervention
3. The Setup/Aha/Habit moments define the success criteria for the feature
4. The funnel percentages define your analytics instrumentation plan — every step must be tracked

### Prioritization: Which Lever First?
Use PNIP to decide:
1. **Adding Use Cases** if users have low breadth (only use one feature/workflow)
2. **Increasing Feature Usage** if users have the right use case but miss high-value features
3. **Increasing Frequency** if users are feature-complete but don't return often enough
4. **Increasing Intensity** if frequency is healthy but session depth is shallow

### Common Mistakes
- **Skipping mindsets and going straight to notifications**: Notifications without mindset understanding create spam. Mindsets tell you *why* users aren't engaging, which determines *what* to say and *when*.
- **Designing for the wrong engagement state**: A feature that moves Casual → Core is not the same as one that moves Core → Power. Design for a specific transition.
- **Tracking only the trigger metric**: "Views of trigger" is not success. Track all the way to "% Reached Habit" — that's the only metric that shows the intervention worked.
- **Jumping to incentives before product**: PNIP order matters. A well-designed product signal beats a badge; a badge beats a promotional email.
